Diagnosed private key mismatch causing certificate creation failure
Diagnosed private key mismatch causing certificate creation failure
Root cause
Every build generates a brand new random private key. The previous build already created a cert on Apple's servers paired with that run's key — which was never saved. Now Apple rejects a second cert (409), and even if it didn't, the new random key would never match the existing cert.

You need one persistent private key stored in Codemagic env vars.

Fix: one-time setup (5 minutes)
Step 1 — Generate the key once on your local machine
bash
openssl genrsa 2048 | base64 | tr -d '\n'
Copy the entire output (a long base64 string).

Step 2 — Add it to Codemagic as an env var
Codemagic UI → your app → Environment variables → add:

Name	Value	Secure
CERT_PRIVATE_KEY_B64	(paste the base64 string)	✅ Yes
Step 3 — Update your YAML signing block
yaml
- name: Set up code signing
  script: |
    # Decode the persistent private key (same key every build)
    echo "$CERT_PRIVATE_KEY_B64" | base64 --decode > /tmp/ios_dist_key.pem
    export CERTIFICATE_PRIVATE_KEY=$(cat /tmp/ios_dist_key.pem)

    keychain initialize

    # Fetch existing cert (matches our stored key) or create if missing
    app-store-connect fetch-signing-files "com.saeed.saman" \
      --type IOS_APP_STORE \
      --certificate-key "$CERTIFICATE_PRIVATE_KEY" \
      --create

    keychain add-certificates

    xcode-project use-profiles
Notice certificates create is gone — fetch-signing-files --create handles everything: it fetches the existing cert that matches your stored key, and creates a provisioning profile if needed.

Why this works now
Same key every build → tool can match the existing cert → no 409
If the cert ever expires/gets revoked, --create makes a new one using the same stored key
No more "Filtered out 1 Signing Certificate for given private key" errors
